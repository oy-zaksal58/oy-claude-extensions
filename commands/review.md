# 코드 리뷰

현재 컨텍스트의 코드를 다음 관점에서 리뷰해주세요.

## 체크리스트

### 1. 보안 (Security)

- [ ] 하드코딩된 시크릿 (API Key, Password)
- [ ] SQL Injection 가능성
- [ ] XSS 취약점
- [ ] 민감한 정보 로깅

### 2. 성능 (Performance)

- [ ] N+1 쿼리 문제
- [ ] 불필요한 반복문
- [ ] 메모리 누수 가능성
- [ ] 비효율적인 알고리즘

### 3. 코드 품질 (Quality)

- [ ] 네이밍 컨벤션 준수
- [ ] 함수 길이 (30줄 이하 권장)
- [ ] 중복 코드
- [ ] 주석의 적절성

### 4. 에러 처리 (Error Handling)

- [ ] 예외 처리 누락
- [ ] 에러 메시지 명확성
- [ ] 로깅 적절성

## 출력 형식

각 항목에 대해 다음 형식으로 정리:

- ✅ **양호**: 문제 없음
- ⚠️ **주의**: 개선 권장 (이유 설명)
- ❌ **심각**: 반드시 수정 필요 (파일:라인 명시)
